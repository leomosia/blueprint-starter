# Plan

Plan is where clarity becomes sequence.

Use this folder to break defined work into milestones, tasks, resources, risks, and timing. Planning should make execution easier, not heavier. A good plan gives enough structure to move without turning the system into administration.

Good planning answers:

- What needs to happen first?
- What resources or information are needed?
- What could block progress?
- What rhythm or timeline makes sense?
- How will progress be tracked?
