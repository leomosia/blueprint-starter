# Blueprint BacklogSys

BacklogSys is the simple holding place for possible work before it becomes actionable.

It is intentionally lighter than TodoSys. BacklogSys can hold rough ideas, future improvements, someday items, possible projects, and unresolved notes that are worth keeping visible but are not ready to become active work yet.

## Files

```text
backlog.md
```

## Principle

BacklogSys collects possibility. TodoSys tracks movement.

Use BacklogSys when something is worth remembering but not yet ready for a story, task, or execution lane.
