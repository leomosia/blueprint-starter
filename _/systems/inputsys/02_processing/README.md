# processing/

## Step 2 — Processing

### What this folder is

`processing/` is a **temporary transformation layer**.

Nothing here is permanent.
Nothing here is “filed for life.”

This is where raw input is **interpreted, clarified, distilled, and routed** into its proper system: domains, fieldnotes, projects, or outputs.

If something lives here too long, it’s a signal of indecision or backlog.

## Core Principle

**Processing does not create the final structure.
Processing prepares items for structure.**

Structure belongs downstream.

## What happens in Processing

### 1. Structuring & Categorization (Pre-Structure)

This phase answers one question only:

**“What is this, really?”**

- **Decoupling & Sorting**
  Break compound thoughts into atomic units.
  One idea = one future destination.

- **Tagging & Metadata**
  Add meaning, not just folders:
  - domain
  - intent
  - status
  - priority
  - confidence / uncertainty
  - source (self, book, conversation, signal)

- **Hard Copy → Digital**
  Transcribe handwritten notes, voice notes, screenshots.
  Fidelity first. Polish later.

No filing decisions yet.

### 2. Refining & Summarization

This phase extracts **signal from noise**.

- **Rewatching & Revisiting**
  Re-consume inputs with intent:
  videos, interviews, courses, conversations.

- **Summarization & Highlighting**
  Reduce content into:
  - distilled notes
  - principles
  - questions
  - frameworks
  - contradictions

- **Cross-Linking**
  Explicitly connect ideas:
  - across domains
  - across time
  - across projects

This is where knowledge becomes usable.

### 3. Filtering & Prioritization

This phase enforces **discipline**.

- **Keep / Delete / Archive**
  Not everything deserves survival.

- **Priority Flagging**
  Mark items as:
  - immediate action
  - slow burn
  - long-term reference
  - dormant

Indecision is a processing failure.

### 4. Automation & System Optimization (Future)

Processing is designed to be **automatable over time**.

- **AI & Script Automation**
  - tagging
  - summarization
  - duplicate detection
  - routing suggestions

- **Workflow Streamlining**
  - inbox → processing → downstream systems
  - minimal human friction
  - maximum clarity per minute spent

Automation assists judgment.
It never replaces it.

## What does NOT belong here

- Final essays
- Published work
- Canonical knowledge
- Long-term archives
- Personal diaries in their final form

If it’s finished, it doesn’t belong here.

## Exit Rule (Non-Negotiable)

Every item in `processing/` must end in **one of three states**:

1. Routed to a domain / fieldnotes / project
2. Archived
3. Deleted

There is no fourth state.

## Mental Model

- **Inbox** → capture without thinking
- **Processing** → think without publishing
- **Output systems** → publish without chaos

Processing is where judgment lives.
