# reviewing/

## Step 6 — Reviewing & Editing

### What this folder is

`reviewing/` is a **quality gate**.

Drafting makes ideas readable.
Reviewing makes them **reliable**.

Nothing passes this step by accident.

## Core Principle

**If it survives review, it deserves attention.**

Reviewing exists to remove weakness, ambiguity, and sloppiness before release.

## What happens in Reviewing

### 1. Structural & Logical Review

This phase tests **thinking integrity**.

- **Organization & Flow Assessment**
  Evaluate whether:

  - arguments build correctly
  - sections flow logically
  - conclusions are earned

- **Gap Identification**
  Look for:

  - missing premises
  - unsupported claims
  - broken reasoning chains

  Weak logic is corrected here.

- **Reordering for Clarity**
  Restructure sections if clarity improves.
  Order follows understanding, not ego.

### 2. Polishing & Precision Editing

This phase sharpens **expression**.

- **Grammar & Syntax Checks**
  Correct:

  - grammar
  - punctuation
  - sentence structure

  Errors dilute authority.

- **Conciseness Enhancement**
  Remove:

  - fluff
  - repetition
  - ornamental language

  Every sentence must earn its place.

- **Style Adaptation**
  Adjust tone and style to match:

  - audience
  - medium
  - purpose

  Precision takes priority over personality.

### 3. Feedback Integration & Final Refinements

This phase introduces **external reality**.

- **Feedback Gathering**
  Collect critique from:

  - peers
  - editors
  - trusted reviewers

  Praise is ignored. Signal is extracted.

- **Revision Iteration**
  Apply feedback deliberately.
  Not all feedback is equal.

- **Accuracy & Reference Checks**
  Verify:

  - facts
  - sources
  - citations
  - claims

  Credibility is non-negotiable.

### 4. Version Control & Categorization

This phase ensures **traceability**.

- **Revision Archiving**
  Preserve meaningful versions to track evolution.
  History matters.

- **Categorization & Tagging**
  Tag final documents for:

  - retrieval
  - reuse
  - cross-system linking

- **Finalization**
  Mark the document as:

  - approved
  - ready for publication
  - ready for application

No further edits without intent.

## What does NOT belong here

- First drafts
- Open-ended exploration
- Structural rethinking
- Emotional writing

If the foundation is unstable, send it back.

## Exit Rule (Non-Negotiable)

A document leaves `reviewing/` only when it is:

1. Logically sound
2. Linguistically clean
3. Factually accurate
4. Purpose-aligned

If it fails any one, it stays.

## Mental Model

- **Drafting** → readable
- **Reviewing** → trustworthy
- **Output** → visible or operational

Reviewing is where standards are enforced.
