# analysis/

## Step 3 — Analysis & Synthesis

### What this folder is

`analysis/` is where processed material is **turned into understanding**.

This is not capture.
This is not cleanup.
This is **thinking under discipline**.

Nothing enters `analysis/` unless it has already survived **Inbox** and **Processing**.

## Core Principle

**Analysis exists to create leverage, not volume.**

If analysis does not change how you see, decide, or act, it has failed.

## What happens in Analysis

### 1. Extraction & Structuring

This phase converts processed material into **atomic knowledge**.

- **Zettelkasten-Style Atomization**
  Break ideas down until each note expresses:
  - one claim
  - one observation
  - one principle
  - one question

  Atomic notes must stand alone.

- **Clean Notes & Structured Summaries**
  Rewrite for:
  - clarity
  - precision
  - future reuse

  This is the first time wording matters.

- **Mind Maps & Visual Structuring**
  Use visuals to:
  - expose relationships
  - surface gaps
  - reveal contradictions

Visuals are thinking tools, not decorations.

### 2. Pattern Recognition & Meaning-Making

This phase answers: **"What is really going on?"**

- **Pattern Extraction**
  Identify:
  - recurring themes
  - incentives and feedback loops
  - cause-and-effect chains

- **Assumption Testing**
  Actively attack:
  - personal bias
  - inherited narratives
  - convenient explanations

  Comfort is a red flag.

- **Comparative Analysis**
  Compare insights against:
  - historical precedents
  - existing frameworks
  - real-world outcomes

Truth survives comparison.

### 3. Idea Development & Knowledge Integration

This phase produces **original thinking**.

- **Concept Synthesis**
  Combine multiple atoms into:
  - models
  - principles
  - theories
  - heuristics

- **System Integration**
  Explicitly connect ideas to:
  - existing fieldnotes
  - strategic domains
  - long-term narratives

No orphan insights allowed.

- **Application Planning**
  Decide where insight lands:
  - decision-making
  - strategy
  - writing
  - systems
  - behavior change

Insight without application is entertainment.

## What does NOT belong here

- Raw notes
- Emotional venting
- Draft publishing
- Personal diaries

If it’s unprocessed or performative, it doesn’t belong.

## Exit Rule (Non-Negotiable)

Every analysis artifact must lead to at least one of:

1. A decision
2. A framework
3. A principle
4. A routed output (writing, system, strategy)

If it doesn’t move something, it’s dead weight.

## Mental Model

- **Inbox** → raw reality
- **Processing** → clarified material
- **Analysis** → structured understanding
- **Output** → public or operational consequence

Analysis is where **signal becomes power**.
