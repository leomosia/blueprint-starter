# Sustain

Sustain is where the system protects continuity.

Use this folder to capture maintenance practices, recovery rhythms, handoff notes, operating standards, and anything needed to keep progress from decaying. Sustain closes the loop by turning useful work into something that can last.

Good sustain work answers:

- What needs to be maintained?
- What should become a repeatable practice?
- What needs rest, recovery, or reset?
- What should be archived?
- What keeps this useful over time?
